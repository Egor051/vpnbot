# Screenshots

The project README references the screenshots below. Drop the image files into this
directory with **exactly these names** and they will render automatically.

| Filename | What to capture |
|---|---|
| `admin-panel.png` | The admin panel main menu (pending requests, users, keys, audit, stats, announcements, diagnostics). |
| `key-issue.png` | The key-creation flow / a delivered Xray or AWG key with its config. |
| `proxy-section.png` | The user-facing «Прокси» section (SOCKS5 / MTProto auto-issue). |
| `warp-panel.png` | The 📡 WARP tunnel admin panel (status, toggle, split routes). |

## Capture tips

- Use a test/staging account so no real tokens, secrets, UUIDs, server IPs, or full
  client configs are visible. **Redact** anything sensitive before committing (the same
  rules as the README "Security Warning" apply to screenshots).
- PNG, reasonable width (≈800–1100 px). Crop to the relevant panel.
- Keep file sizes modest (compress if needed) so the repo stays light.
- Light or dark Telegram theme is fine; be consistent across the set if possible.
